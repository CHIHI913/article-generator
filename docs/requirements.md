# 記事生成アプリ 要件定義書

作成日: 2025-06-15
作成者: openhands
更新履歴: 2025-06-15 初版作成

---

## 1. 目的
ユーザーが任意の概要を入力するだけで、複数の定義済みフォーマットに沿った高品質な記事を自動生成できる Web アプリケーションを提供する。Vercel の AI SDK を活用し、ユーザー自身が AI モデルを追加・切替可能とすることで、高い拡張性とカスタマイズ性を実現する。

## 2. 背景・課題
- **コンテンツ制作コストの増大**: 記事作成に要する時間・コストが増大している。
- **AI 活用ニーズ**: GPT-4o などの生成 AI を活用し、下書きや構成を高速生成したい。
- **フォーマット統一**: SEO 用、ニュース用など目的に応じた記事フォーマットを手軽に切替えたい。
- **モデル選択**: 企業・個人で学習済みの独自モデルや、最新の公開モデルを任意に利用したい。

## 3. 用語定義
|用語|定義|
|---|---|
|フォーマット|見出し構成やメタ情報 (タイトル、リード文、タグ 等) を含む記事テンプレート|
|モデル|Vercel AI SDK で利用可能な AI 言語モデル (OpenAI, Anthropic など) またはユーザー独自モデル|
|プロンプト|AI に渡す指示文。フォーマットと概要から動的生成される|

## 4. 想定ユーザー
- ブロガー / メディア編集者
- マーケティング担当者 (SEO 用記事)
- 社内情報共有を効率化したい企業ユーザー

## 5. システム概要
```
+------------+        +-------------------+        +--------------------------+
|   ブラウザ  | -----> | Next.js UI (SSR)  | -----> | Vercel AI SDK (Edge)     |
+------------+        |  - 入力フォーム   |        |  - OpenAI 等の LLM       |
                     |  - 記事プレビュー |        |  - User Provided Models  |
                     +-------------------+        +--------------------------+
                            |                                     |
                            v                                     v
                     +-------------------+                +-------------+
                     |  データストア     |<---------------| モデル管理API|
                     |  - 生成履歴       |                +-------------+
                     |  - ユーザー設定   |
                     +-------------------+
```

## 6. 機能要件
### 6.1 記事生成
- R1-01: ユーザーは記事の"概要"(最低 10 文字)を入力できる。
- R1-02: 概要を基に、選択したフォーマットで記事ドラフトを生成する。
- R1-03: 生成処理は Vercel AI SDK を用い、ストリーミングでプログレスを表示する。
- R1-04: 生成後、ユーザーは編集可能なプレビューを確認できる。

### 6.2 フォーマット管理
- R2-01: 管理画面でフォーマットを CRUD できる (管理者権限)。
- R2-02: フォーマットは JSON/YAML で定義し、見出し順序・装飾ルールを保持する。
- R2-03: デフォルトで「ブログ」「ニュース」「SEO ランディング」の 3 種を提供する。

### 6.3 モデル管理
- R3-01: ユーザーは利用可能な AI モデル一覧を閲覧できる。
- R3-02: API キーまたはエンドポイントを入力して独自モデルを登録できる。
- R3-03: モデルごとにトークンコスト・最大入力長を表示する。
- R3-04: 記事生成時にモデルを選択できる。

### 6.4 履歴・保存・エクスポート
- R4-01: 生成した記事を履歴として保存し、再編集できる。
- R4-02: Markdown / HTML / JSON 形式でダウンロード可能とする。
- R4-03: SaaS 連携 (Notion, WordPress) は将来拡張で対応する。

### 6.5 ユーザー管理 (オプション初期 MVP では Guest 対応)
- R5-01: SNS OAuth (GitHub / Google) によるログイン。
- R5-02: ロール: ユーザー / 管理者。管理者のみフォーマット CRUD 可。

## 7. 非機能要件
|分類|ID|内容|指標|
|----|--|----|----|
|パフォーマンス|NF-01|生成リクエストは平均 15 秒以内に完了|P95 < 15s|
|スケーラビリティ|NF-02|同時 100 リクエストを捌ける|水平スケール|
|UX|NF-03|ストリーミングにより 3 秒以内に最初のレスポンス|
|セキュリティ|NF-04|API キーは暗号化して保存|
|可観測性|NF-05|OpenTelemetry でトレーシング|
|ブラウザ互換|NF-06|最新 2 版の Chrome, Safari, Edge|

## 8. 制約条件
- ホスティング: Vercel (Edge Functions 利用可)
- 使用言語: TypeScript, Next.js 14 (app router)
- AI SDK: `@vercel/ai` を必須利用
- モデル登録時、ユーザー責任でライセンス遵守

## 9. 画面一覧 (概要)
|画面ID|名称|主要要素|
|---|---|---|
|SCR-01|記事生成画面|概要入力フォーム、フォーマット選択、モデル選択、生成ボタン、プレビュー|
|SCR-02|フォーマット管理|一覧、作成/編集ダイアログ、JSON エディタ|
|SCR-03|モデル管理|モデル一覧、登録フォーム、テストボタン|
|SCR-04|履歴一覧|生成日時、フォーマット、モデル、再編集 / エクスポート|
|SCR-05|ログイン|OAuth ボタン|

## 10. ユースケース
1. ユーザーはログインなしで SCR-01 にアクセスし、概要を入力する。
2. "フォーマット"=ブログ、"モデル"=gpt-4o を選択し、生成ボタンをクリック。
3. 数秒でストリーミングが開始し、全体が生成されプレビュー欄に挿入される。
4. 気になる部分をブラウザ上で修正し、Markdown でダウンロード。

## 13. リスクと対応
|リスク|対応策|
|---|---|
|モデル利用料金の高騰|料金上限を設定、トークン使用量を可視化|
|生成品質のばらつき|フォーマットごとにシステムプロンプトをチューニング|
|不適切な内容生成|OpenAI Moderation API によるフィルタリング|
