# プロジェクトコンテキスト

## プロジェクト概要
**記事生成アプリケーション**  
ユーザーが任意の概要を入力するだけで、複数の定義済みフォーマットに沿った高品質な日本語記事を自動生成するWebアプリケーション。

## 技術スタック
- **フレームワーク**: Next.js 15 (App Router)
- **言語**: TypeScript
- **AI SDK**: Vercel AI SDK (@ai-sdk/openai, @ai-sdk/react)
- **スタイリング**: Tailwind CSS v4
- **デプロイ**: Vercel (Edge Functions)
- **AIモデル**: OpenAI GPT-4o

## プロジェクトの制約・決定事項

### アーキテクチャ制約
- **Edge Runtime必須**: レスポンス速度向上のため全APIルートでEdge Runtimeを使用
- **ストリーミング必須**: 生成時間が長いため、リアルタイムストリーミングで体験向上
- **日本語特化**: システムプロンプト・UIすべて日本語対応

### 技術的制約
- **ストリーミングプロトコル**: `useCompletion`の`streamProtocol`と API側の`to*StreamResponse()`は必ず対応させる
  - `text` ↔ `toTextStreamResponse()`
  - `data` ↔ `toDataStreamResponse()`
- **概要最小文字数**: 10文字以上（品質確保のため）
- **最大トークン**: 2048トークン（コスト管理）

### ビジネス制約
- **MVP段階**: 認証機能なし、ゲストアクセスのみ
- **3つの基本フォーマット**: ブログ、ニュース、SEOランディングページ
- **OpenAI固定**: 現時点ではGPT-4oのみ、将来的にマルチモデル対応予定

## 重要な設計思想
1. **シンプルさ優先**: MVPとして最小限の機能で開始
2. **拡張性確保**: フォーマットシステムで将来の機能拡張に対応
3. **日本語最適化**: 日本語記事生成に特化したプロンプト設計
4. **ユーザー体験**: ストリーミングによる待機時間の体感短縮
